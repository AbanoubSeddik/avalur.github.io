Git is a distributed version control system, designed with a focus on speed and efficiency of interaction.

Distributed means:

  • each developer has a complete, local repository
  • technically the central repo is not different from the local repos
  • easy offline usage
  • easy to branch a project

Examples: Github, Gitlab, Mercurial

  • Created 2005 for Linux Kernel Development
  • Used for Linux, Android, Eclipse, ...
  • Used everywhere: Google, JetBrains, SAP, Qualcomm, Nvidia, Garmin, X, etc.

Basic Workflow

  1. Clone the repository:

    $> git clone git@github.com:jetbrains-academy/computer-science-basics-with-python.git

  2. Make changes using IDEA/VS Code/vim/emacs/Cursor/Zed...
  3. Save changes: $> git add (file)
  4. Review changes: $> git status / git diff
  5. Commit changes: $> git commit

Introduction to CI/CD

  • CI: Continuous Integration
  • CD: Continuous Deployment/Delivery
  • Automate the software development lifecycle
  • Enable faster and more reliable software releases

Benefits of CI/CD

  • Early detection of defects
  • Reduced integration issues
  • Faster time to market
  • Improved collaboration and transparency

GitHub Actions

  • CI/CD platform integrated with GitHub
  • Automate workflows directly from your repository
  • Trigger actions on events like pushes, pull requests, and more
  • Supports custom scripts and community-contributed actions

Basic Workflow of GitHub Actions

  1. Create a workflow file: .github/workflows/ci.yml
  2. Define triggers (events): on: push
  3. Specify jobs and steps
  4. Use pre-built actions or write custom scripts
  5. View results in the GitHub Actions tab

Sample GitHub Actions Workflow


					name: CI

					on: [push, pull_request]

					jobs:
					  build:
						runs-on: ubuntu-latest
						steps:
						- uses: actions/checkout@v3
						- name: Set up Python
						  uses: actions/setup-python@v4
						  with:
							python-version: '3.9'
						- name: Install dependencies
						  run: pip install -r requirements.txt
						- name: Run Tests
						  run: pytest

Advantages of GitHub Actions

  • Seamless integration with GitHub repositories
  • Easy to set up and customize
  • Supports multiple languages and platforms
  • Community-driven with a marketplace of actions

CI/CD Best Practices

  • Commit code frequently
  • Write tests for your code
  • Automate builds and testing
  • Monitor and analyze pipeline performance
  • Keep the pipeline fast and efficient
